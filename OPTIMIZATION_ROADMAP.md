# Mini Desk Tool 更新优化路线

本文档用于规划 Mini Desk Tool 后续功能优化。路线重点围绕交互体验、日常使用效率、窗口行为、数据管理和发布流程展开。

## 产品定位

Mini Desk Tool 的核心方向是：桌面边缘的轻量 Launchpad。

目标不是把功能堆满，而是让用户在桌面上更快完成三件事：

- 找到常用网站、文件、文件夹和应用。
- 快速整理图标和分组。
- 让面板稳定贴在桌面边缘，随用随取，不打扰当前工作。
- 现代化UI, 界面简洁为主，交互体验必须丝滑

## 优先级原则

- 高频优先：每天都会用到的搜索、最近打开、拖拽、分组管理优先。
- 低打扰优先：窗口行为必须稳定，不能因为吸附、缩放、隐藏影响使用。
- 可恢复优先：涉及数据变更的功能，需要有导入、导出、备份或撤销方案。
- 小步发布：每个版本只解决一个明确主题，避免一次性改动过大。
- 每次完成一个版本都执行压缩上下文

## v1.2.0：交互效率

### 目标

减少查找和整理成本，让面板在图标数量变多后依然好用。

### 功能清单

- 新增顶部搜索入口，输入关键词后实时过滤图标。
- 搜索支持匹配名称、描述、网址、文件路径和分组名。
- 按 `Enter` 打开搜索结果第一项。
- 新增最近打开区域，记录最近 5 到 10 个项目。
- 分组支持折叠和展开。
- 拖拽图标时显示插入位置和目标分组高亮。
- 不可拖入的位置显示轻提示。

### 验收标准

- 50 个图标以上时，搜索仍能快速响应。
- 搜索结果为空时，有明确空状态。
- 最近打开不会重复记录同一个项目。
- 分组折叠状态刷新后仍能保留。
- 拖拽时能清楚看出图标会落到哪里。

### 开发流程

1. 扩展状态模型，加入最近打开和分组折叠状态。
2. 实现搜索过滤逻辑，先不改变原有数据结构。
3. 改造渲染层，让搜索、最近打开、普通分组可以共存。
4. 优化拖拽反馈样式和边界提示。
5. 补 Playwright 测试：搜索、最近打开、分组折叠、拖拽反馈。

## v1.3.0：管理能力

### 目标

让用户能更轻松地导入、批量整理和维护大量图标。

### 功能清单

- 支持扫描桌面快捷方式并批量导入。
- 支持扫描开始菜单快捷方式并批量导入。
- 图标支持多选。
- 多选后支持批量移动分组、删除、调整大小。
- 支持把本地图片拖到图标上作为自定义图标。
- 本地文件、文件夹、应用图标缓存到本地，减少重复解析。

### 验收标准

- 批量导入前能预览条目并取消不需要的项目。
- 批量删除前必须二次确认。
- 图标缓存失败时不影响项目打开。
- 重复导入同一路径时能提示或自动去重。

### 开发流程

1. 设计批量导入数据结构和预览弹窗。
2. Rust 侧增加桌面和开始菜单扫描命令。
3. 前端实现批量选择和批量操作状态。
4. 增加图标缓存目录和失效策略。
5. 补导入、去重、批量删除、图标缓存测试。

## v1.4.0：窗口体验

### 目标

让面板更像真正贴在桌面边缘的工具，而不是一个普通小窗口。

### 功能清单

- 支持自动隐藏到屏幕边缘。
- 鼠标靠近边缘时自动唤出。
- 支持设置吸附边缘：左、右、上、下。
- 支持设置吸附距离和唤出延迟。
- 支持全局快捷键显示或隐藏面板。
- 设置面板增加窗口行为区域。

### 验收标准

- 自动隐藏后不会遮挡正常桌面操作。
- 靠边唤出在多显示器环境下行为可预测。
- 全局快捷键冲突时有明确提示。
- 禁用自动隐藏后，窗口恢复普通吸附行为。

### 开发流程

1. 梳理 Tauri 窗口事件和多显示器坐标逻辑。
2. 先实现手动隐藏和快捷键唤出。
3. 再实现边缘自动隐藏和鼠标唤出。
4. 设置项入库，保证重启后行为一致。
5. 在 100%、125%、150% 显示缩放下手动验证。

## v1.5.0：稳定与备份

### 目标

降低用户误操作和数据丢失风险。

### 功能清单

- 支持自动备份到用户选择的目录。
- 支持保留最近若干份备份。
- 设置面板显示最近一次备份时间。
- 导入数据前自动创建恢复点。
- 添加错误提示中心，用于展示导入、图标解析、快捷方式解析失败。

### 验收标准

- 备份失败不影响主功能使用。
- 导入错误数据时不会覆盖当前可用数据。
- 用户可以从最近备份恢复。
- 错误提示可关闭，不会长期占用界面。

### 开发流程

1. 定义备份文件格式和版本字段。
2. 增加备份目录选择和权限检查。
3. 导入流程加入恢复点。
4. 实现错误提示中心。
5. 补导入失败、恢复、备份轮转测试。

## v1.6.0：个性化

### 目标

满足不同使用场景，让面板可以适配工作、学习、娱乐等多种布局。

### 功能清单

- 支持多配置文件，例如工作、个人、临时项目。
- 支持布局预设：紧凑、标准、宽屏。
- 支持主题色和透明度设置。
- 支持隐藏分组标题、隐藏最近打开等显示选项。

### 验收标准

- 切换配置时不会丢失当前配置数据。
- 布局预设不会破坏用户自定义宽度和图标大小。
- 主题设置在浅色和深色桌面背景上都有足够可读性。

### 开发流程

1. 抽象当前状态为 profile 结构。
2. 实现配置切换和默认配置迁移。
3. 增加布局预设和主题变量。
4. 补配置切换、迁移、主题持久化测试。

## 通用开发流程

每个版本按以下流程推进：

1. 确认需求范围：列出本版本必须做和暂不做的内容。
2. 写交互草案：明确入口、状态、空状态、错误状态。
3. 拆实现任务：前端状态、渲染、Rust 命令、测试分别拆开。
4. 先做最小可用版本：保证主流程跑通。
5. 补边界场景：空数据、大量数据、失败、取消、重复操作。
6. 跑验证：至少执行 `cargo test`、`npm run build:renderer`、相关 Playwright 测试。
7. 更新文档：README、版本说明、使用说明同步更新。
8. 构建安装包：执行 `npm run build`。
9. 发布 GitHub Release：上传安装包并写清楚变更点。

## 每版发布检查清单

- 版本号已同步：`package.json`、`package-lock.json`、`src-tauri/Cargo.toml`、`src-tauri/tauri.conf.json`。
- README 已更新。
- 中文 README 已更新。
- 关键功能有 Playwright 测试或手动验证记录。
- `cargo test` 通过。
- `npm run build:renderer` 通过。
- `npm run build` 成功生成 NSIS 安装包。
- 安装包文件名和版本号一致。
- Git commit 已创建。
- `main` 已推送。
- GitHub Release 已创建并上传安装包。
