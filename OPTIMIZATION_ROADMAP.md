# Mini Desk Tool 更新优化路线

本文档用于规划 Mini Desk Tool 的后续功能优化。路线重点围绕交互体验、日常使用效率、窗口行为、数据安全、个性化和发布流程展开。

## 产品定位

Mini Desk Tool 的核心方向是：桌面边缘的轻量 Launchpad。

目标不是把功能堆满，而是让用户在桌面上更快完成三件事：

- 找到常用网站、文件、文件夹和应用。
- 快速整理图标和分组。
- 让面板稳定贴在桌面边缘，随用随取，不打扰当前工作。
- 保持现代、轻量、可控的界面体验。

## 优先级原则

- 高频优先：每天都会用到的搜索、最近打开、拖拽、分组管理优先。
- 低打扰优先：窗口吸附、缩放、隐藏、唤出必须稳定，不能干扰桌面操作。
- 可恢复优先：涉及数据变更的功能，必须有导入、导出、备份或撤销方案。
- 小步发布：每个版本只解决一个明确主题，避免一次性改动过大。
- 验证优先：每个版本完成后都需要构建、测试、记录结果，再等待下一步指令。

## v1.1.2：交互效率

### 目标

减少查找和整理成本，让面板在图标数量变多后仍然好用。

### 已完成

- 新增顶部搜索入口，支持实时过滤图标。
- 搜索支持匹配名称、描述、网址、文件路径和分组名称。
- 搜索功能可在设置中开启或关闭。
- 搜索栏支持选择搜索引擎，按 `Enter` 调用外部搜索。
- 新增最近打开区域，并支持在设置中开启或关闭。
- 最近打开支持实时刷新、分页显示和单独移除记录。
- 最近打开图标固定以 `1x1` 尺寸展示，不影响原图标尺寸。
- 分组支持折叠和展开。
- 拖拽图标时显示插入位置和目标分组高亮。

### 验收标准

- 搜索结果为空时有明确空状态。
- 最近打开不会重复记录同一个项目。
- 最近打开删除按钮只删除最近记录，不删除原位置图标。
- 分组折叠状态刷新后仍能保留。
- 拖拽时能清楚看出图标会落到哪里。

## v1.1.3：管理能力

### 目标

让用户能更轻松地导入、整理和维护大量图标。

### 已完成

- 支持扫描桌面快捷方式并批量导入。
- 批量工具条支持多选图标。
- 支持全选当前分组和全选所有图标。
- 支持批量移动到分组。
- 支持批量调整大小。
- 支持批量删除。
- 暂时屏蔽描述过长的导入入口，保留更精简的使用路径。

### 验收标准

- 批量工具条入口清晰，选中图标后能立即出现。
- 批量移动、改大小、删除不会影响未选中的图标。
- 第一组为空时仍保留分组。
- 界面至少保留一个分组，即使为空也不清除。

## v1.1.4：窗口体验

### 目标

让面板更像真正贴在桌面边缘的工具，而不是普通小窗口。

### 已完成

- 设置面板增加“窗口行为”区域。
- 支持失焦自动隐藏实验开关。
- 支持自动隐藏后的边缘唤出。
- 支持设置吸附边缘：左、右、上、下。
- 支持设置吸附距离和唤出延迟。
- 支持全局显示/隐藏快捷键开关和快捷键配置。
- 快捷键注册失败时有明确提示。
- 修复窗口调整大小后界面消失的问题。
- 最小窗口宽度调整为 `300px`。
- 检查拖拽窗口吸附逻辑，避免和自动隐藏逻辑冲突。

### 验收标准

- 自动隐藏后不会遮挡正常桌面操作。
- 拖拽窗口后只在拖拽结束时触发吸附。
- 禁用自动隐藏后，窗口恢复普通吸附行为。
- 多显示器和不同缩放比例下行为可预期。

## v1.1.5：稳定与备份

### 目标

降低误操作和数据丢失风险。

### 已完成

- 设置面板增加“备份与恢复”区域。
- 支持选择备份目录。
- 支持设置备份保留数量。
- 支持立即备份。
- 数据变更后支持低打扰自动备份。
- 导入数据前自动创建恢复点。
- 支持恢复导入前状态。
- 添加错误提示中心，用于展示导入、导出、备份、恢复和快捷方式解析失败。

### 验收标准

- 备份失败不影响主功能使用。
- 导入错误数据时不会覆盖当前可用数据。
- 用户可以通过导入备份文件恢复数据，也可以恢复导入前状态。
- 错误提示可关闭，不会长期占用界面。

## v1.1.6：个性化

### 目标

满足不同使用场景，让面板适配工作、学习、娱乐等多种布局。

### 已完成

- 支持多配置文件切换。
- 支持从当前配置复制新建配置。
- 支持配置重命名。
- 支持紧凑、标准、宽屏和自定义布局预设。
- 支持主题选择。
- 支持面板透明度设置，并让透明度影响背景、工具条和图标卡片。
- 支持隐藏分组标题。
- 支持隐藏最近打开。
- 支持隐藏图标名称。
- 设置面板按功能分组优化。
- 搜索引擎下拉栏样式优化。
- 搜索框只保留一个清除按钮，避免出现两个 `X`。

### 验收标准

- 切换配置时不会丢失当前配置数据。
- 布局预设不会破坏用户自定义宽度和图标大小。
- 主题设置在浅色和深色桌面背景上都具有足够可读性。
- 透明度变化肉眼可见。
- 设置项按功能分组后更容易查找。

## v1.2.1：搜索增强

### 目标

把搜索从“过滤列表”升级为更完整的启动入口。

### 功能方案

- 搜索结果按匹配来源排序：名称优先，其次描述、网址、路径、分组。
- 搜索结果中高亮匹配关键词。
- 支持搜索历史，下次打开时可快速选择最近搜索词。
- 支持搜索栏快捷前缀，例如 `g 关键词` 使用 Google，`b 关键词` 使用百度。
- 支持未命中本地图标时，直接使用当前搜索引擎搜索。
- 支持清空搜索历史。

### 验收标准

- 本地图标匹配优先于外部搜索。
- 搜索历史不会记录空内容。
- 搜索历史可关闭或清空。
- 前缀搜索不会破坏现有搜索引擎设置。

### 开发流程

1. 扩展搜索运行时状态，增加搜索历史和匹配来源。
2. 改造搜索结果构建逻辑，记录命中字段和排序分数。
3. 优化搜索栏交互，增加历史选择和关键词高亮。
4. 增加前缀搜索解析逻辑。
5. 补充搜索排序、历史、前缀搜索和外部搜索测试。

## v1.2.2：图标资产管理

### 目标

提升图标加载质量，减少网络图标失败、模糊或重复解析的问题。

### 功能方案

- 增加图标刷新按钮，可对单个图标重新获取候选图标。
- 支持批量刷新选中图标。
- 支持本地图标缓存状态展示。
- 支持清理无效图标缓存。
- 支持为文件、文件夹、应用生成更稳定的本地图标快照。
- 图标加载失败时显示更明确的占位图标和失败原因。

### 验收标准

- 图标刷新失败不影响项目打开。
- 自定义图标优先级高于自动图标。
- 清理缓存不会删除用户自定义图标。
- 批量刷新过程中界面不阻塞。

### 开发流程

1. 梳理当前图标来源优先级：自定义、本地快捷方式、网站 favicon、回退图标。
2. 为图标增加缓存元信息：来源、更新时间、失败原因。
3. 实现单个刷新和批量刷新入口。
4. 增加缓存清理命令。
5. 补充图标刷新、失败回退、缓存清理测试。

## v1.2.3：分组与整理增强

### 目标

让大量图标的整理更轻松，减少手工拖拽成本。

### 功能方案

- 支持按名称、最近打开时间、添加时间、类型排序。
- 支持分组内一键整理空位。
- 支持重复项检测，按网址、路径或标题提示可能重复。
- 支持自动分组建议，例如开发、设计、视频、办公。
- 支持空分组保留策略：始终保留、手动清理、自动隐藏。
- 支持分组颜色或轻量标记。

### 验收标准

- 排序前给出确认或可撤销提示。
- 自动分组只提供建议，不直接覆盖用户分组。
- 重复项检测不会误删数据。
- 空分组策略不会让界面变成无分组状态。

### 开发流程

1. 增加排序和整理动作入口。
2. 实现重复项扫描函数，只生成建议列表。
3. 实现自动分组建议，不直接修改数据。
4. 增加分组标记字段和渲染样式。
5. 补充分组排序、重复项、空分组策略测试。

## v1.2.4：快捷动作中心

### 目标

让每个图标不只可以打开，还可以执行更丰富的快捷动作。

### 功能方案

- 右键菜单增加“复制链接/路径”。
- 支持“以管理员身份打开”入口，仅对本地应用显示。
- 支持“打开所在文件夹”。
- 支持“在浏览器中打开”和“复制搜索关键词”。
- 支持给图标配置备注或快捷命令。
- 支持常用动作置顶。

### 验收标准

- 不同类型项目只展示可用动作。
- 复制动作成功后有轻量提示。
- 本地路径动作失败时有错误提示，不影响其他功能。
- 右键菜单不会过长，低频动作可折叠。

### 开发流程

1. 抽象项目类型识别：网址、文件、文件夹、应用。
2. 为右键菜单增加动作注册表。
3. Rust 侧补充打开所在文件夹、管理员打开等命令。
4. 增加动作成功和失败提示。
5. 补充不同类型项目的菜单和动作测试。

## v1.2.6：数据安全增强

### 目标

进一步提升备份可理解性和恢复可控性。

### 功能方案

- 备份列表展示创建时间、项目数量、配置数量和文件大小。

- 撤销失败时不影响当前数据。
- 变更历史不记录敏感文件内容，只记录操作摘要。

### 开发流程

1. 扩展备份索引文件，记录备份摘要。
2. 实现备份对比函数，生成恢复预览。
3. 增加备份备注和锁定字段。
4. 增加高风险操作历史。
5. 补充恢复预览、锁定备份、撤销操作测试。

## 通用开发流程

每个版本按以下流程推进：

1. 确认需求范围：列出本版本必须做和暂不做的内容。
2. 写交互草案：明确入口、状态、空状态、错误状态。
3. 拆实现任务：前端状态、渲染、Rust 命令、测试分别拆开。
4. 先做最小可用版本：保证主流程跑通。
5. 补边界场景：空数据、大量数据、失败、取消、重复操作。
6. 跑验证：至少执行 `cargo test`、`npm run build:renderer`、相关 Playwright 测试。
7. 更新文档：README、版本说明、使用说明同步更新。
8. 构建安装包：执行 `npm run build`。
9. 按用户指令决定是否提交、推送或发布 GitHub Release。

## 每版发布检查清单

- 版本号已同步：`package.json`、`package-lock.json`、`src-tauri/Cargo.toml`、`src-tauri/tauri.conf.json`。
- README 已更新。
- 中文 README 已更新。
- `OPTIMIZATION_ROADMAP.md` 已更新。
- 关键功能有 Playwright 测试或人工验证记录。
- `cargo test` 通过。
- `npm run build:renderer` 通过。
- `npm run build` 成功生成安装包。
- 安装包文件名和版本号一致。
- Git commit 已创建。
- `main` 已推送。
- GitHub Release 按需创建并上传安装包。
